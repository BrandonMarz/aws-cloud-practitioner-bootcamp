## Purpose

This lab introduces students to AWS compute services and infrastructure thinking.

You will learn how AWS runs:

- Applications
- Servers
- Workloads
- Automation systems

This is your first step into real compute engineering.

---

## Learning Philosophy

This lab is **NOT** about:

- Memorizing buttons
- Copying tutorials
- Launching random servers

This lab **IS** about:

- Infrastructure awareness
- Cloud engineering thinking
- Security-conscious deployment
- Understanding compute decisions

---

## Required AWS Services

Students will work with:

- Amazon EC2
- Security Groups
- Instance Types
- AWS Lambda

---

## Important Rules

- Never leave unnecessary ports open
- Never expose infrastructure carelessly
- Always validate access controls
- Always stop unused resources after labs
- Think about cost and security together

---

## Core Learning Outcomes

By completing this lab, students should understand:

- How compute infrastructure works in AWS
- How EC2 instances are deployed and managed
- How security groups control network access
- How serverless computing works with Lambda
- How infrastructure decisions affect cost and security
- Basic cloud engineering best practices

---

## Recommended Mindset

Think like an infrastructure engineer:

- Secure first
- Automate where possible
- Minimize unnecessary exposure
- Monitor resource usage
- Understand why systems are built a certain way
